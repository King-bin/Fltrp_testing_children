# 《外研定级测·幼儿版》产品需求文档 \(PRD\) V2\.0

# 项目背景

外研定级测是专为中国英语学习者打造的定级测试系统 。目前已上线“小初版”（S1\-S9级别）。小初版主要是通过H5去进行实现，有两个使用方式，一种是纯H5的端口，包含了注册登录、信息选择、测评，还有报告，然后一种是小程序里，小程序嵌套H5，并增加小程序一键登录。



现在需要开发幼儿版，针对 3\-7 岁幼儿开发独立的功能模块，通过游戏化寻宝剧情，精准定位幼儿的英语启蒙水平（K1\-K3级别，定级会拓展到S1\-S4，因为有的还是水平比较高，已经达到了小学的阶段）。



目前幼儿版计划是一个单独的产品，先以H5的形式实现，后续考虑把H5嵌套到小程序中（两种嵌套方式，一种是单独申请一个幼儿版的小程序，还有一种是嵌套在现有的小初版中，在初版开发时，要考虑后续的嵌入）  。



幼儿班整体是强制横屏。

## 第二章：UI/UX 视觉与交互标准

### 2\.1 视觉风格定义 \(Visual Identity\)

幼儿版视觉风格定位于“**沉浸式自然绘本感**”，增强色彩饱和度与图形圆润度。

#### 2\.1\.1 色彩系统

- **森林主题色**：

    - **主色（森林绿 \#2E7D32）**：用于背景基础色、进度条填充，呼应“寻宝”主题 。  

    - **辅助色（草绿 \#81C784）**：用于次要装饰或非选中状态。

- **交互强调色**：

    - **核心色（活力橙 \#FF8F00）**：用于底部录音键、确认键，确保在绿色背景中具备最高辨识度。  

    - **反馈色（亮黄色 \#FFEB3B）**：用于选中状态的边框高亮或得分激励。

- **功能色**：

    - **确认勾选（绿色 \#4CAF50）**：对应“$\\checkmark$”选项 。  

    - **取消/错误（蓝色 \#2196F3）**：对应“$\\times$”选项，替代红色以减轻幼儿的挫败感 。  

#### 2\.1\.2 图形与质感

- **高圆角设计**：所有卡片（题目展示、选项容器）的 `border-radius` 统一设定为 `32px` 或 `40px`，消除锐利感。

- **微拟物按钮**：交互按钮需带有轻微的 `box-shadow`（投影）和 渐变色 ，营造“积木”触感。  

- **背景表现**：

    - **基础包**：深浅交替的森林绿植插画 。  

    - **增强包**：带有帐篷、气球元素的嘉年华乐园场景 。  

---

### 2\.2 物理布局参数 \(Layout Specification\)

强制横屏模式下，页面采用“**三段式、非对称**”布局，高度 $H=100vh$，宽度 $W=100vw$。

---

### 2\.3 IP 角色动态反馈规范

IP 角色不仅是装饰，需承担“教学指引”职能：

- **画外音同步**：

    - 播放“画外音”（Voice\-over）时，IP 需有开口动画及指向右侧的手势 。  

    - 说话期间，IP 头顶出现“声音波纹”图标。

- **交互激励**：

    - **提交反馈**：每道题点击下一题/提交后，IP 随机做出激励动作和音效 。 

    - **待机逻辑**：超过 10 秒未操作，IP 做出“思考”或“挥手”动作，引导幼儿关注题目。

---

### 2\.4 交互微动效标准

- **点击反馈**：按钮点击时下沉 $4px$，并伴随清脆的“水滴”音效。

- **选中反馈**：选项卡片外圈出现 $6px$ 的黄色发光边框。

- **转场过渡**：

    - **任务切换**：屏幕中心出现横向平移的地图卷轴动效 。    

---

### 2\.5 字体规范

- **中文字体**：`PingFang SC` / `Source Han Sans`，加粗，字号不小于 `32px`。

- **英文字体**：`Lexend` 或 1\.0 版内置圆润英文字体，字号 `48px - 72px`。

- **颜色**：题目文本使用深炭灰色 \(\#333333\)，避免使用纯黑色。

## 交互状态机 \(Interaction State Machine\)

### 阶段一：入场与锁定态 \(Entering \& Locked\)

- **触发**：组件加载。

- **动作（仅每组第一道题自动触发该动作）**：

    1. 左下角 IP 播放画外音（VO）。一组题共用一个画外音。 仅本组第一题加载时自动播放一次。 

    2. **强制锁定**：VO 播放期间，底部 `✓`、`×` 及 “下一题”或“提交”按钮 按钮置灰，禁用所有点击事件 。  

### 阶段二：待机与提示态 \(Idle \& Prompting\)

- **动作**：画外音结束或组件加载完成。

- **核心逻辑**：

    1. 自动触发左上角（或居中）题干音频播放。连续播放 **2 遍** 后停止 。  

    2. **音频键状态切换**：

        - **播放态**：播放中，图标呈现动态波纹。点击可停止播放 。  

        - **停止态**：播放结束或手动停止，图标变静态。点击可重新播放（播放 1 遍后自动停止） 。  

### 阶段三：作答态 \(Answering\)

- **动作**：幼儿点击 `✓` 或 `×`。

- **微动效反馈**：

    - **触感反馈**：点击瞬间，按钮执行 `scale(0.9)` $\\rightarrow$ `scale(1.1)` $\\rightarrow$ `scale(1.05)` 的弹簧动画，耗时 `300ms` 

    - **视觉高亮**：黄色（\#FFEB3B）发光边框 ，未选中项透明度保持 `0.7` 。  

- **完成信号**：作答后，右下角“下一题”按钮由【置灰】变为【高亮呼吸态】 。  

### 阶段四：放行态 \(Completed\)

- **容错处理**：在点击“下一题”前，幼儿可随意更改 `✓/×` 选择。点击“下一题”时，系统记录最终结果并执行过场动画 。  

---

## 异常处理与防错 \(Edge Cases\)

- **静默引导**：若阶段二音频停止后 10 秒无操作，IP 角色触发挥手动作并重播 画外音（VO） 吸引注意力 。  

- **点击防抖**：针对幼儿快速拍打，所有交互按钮设置 `300ms` 的点击防抖逻辑 。

## 交互状态机 \(Interaction State Machine\)

### 阶段一：入场与锁定态 \(Entering \& Locked\)

- **触发**：组件加载。

- **动作（仅每组第一道题自动触发该动作）**：

    1. 左下角 IP 播放画外音（VO）。一组题共用一个画外音。 仅本组第一题加载时自动播放一次。 

    2. **强制锁定**：VO 播放期间，操作区所有按钮及右下角“下一题”或“提交”按钮置灰，**完全禁用点击事件**。

### 阶段二：待机与提示态 \(Idle \& Prompting\)

- **动作**：画外音播放结束。

- **连环指引逻辑**：

    1. 自动触发**左侧题干音频**播放 2 遍。同时出现动态“小手”指向该按钮。（仅听音跟读和听音说话题型有）

    2. 播放结束后，“小手”平滑移向**中间录音按钮**并闪烁，提示幼儿开口。

    3. **左侧题干音频**自动播放后，可再次点击进行播放，播放1次停止

### 阶段三：作答态 \(Recording \& Generation\)

- **触发**：幼儿点击中间录音按钮，播放“叮”提示音。

- **反馈**：激活声波光环，按钮最边缘有圆环动态倒计时，倒计时15S，IP 做出倾听动作。录音态强排他性。一旦进入 Recording 状态，不仅 BGM 闭音，同时必须将左侧的“原音”按钮和右下角的“下一题”置灰且不可点击 ，防止示范音二次混入录音流导致声通引擎判定失败 。

- **生成**：用户在倒计时内点击录音按钮或者倒计时结束，录音停止，播放“叮”提示音。

- **激活**：**右侧“用户音频”按钮瞬间亮起**，支持点击回放。

### 阶段四：放行态 \(Completed\)

- **联动**：录音成功结束（即生成了“用户音频”）后，右下角“下一题”由置灰切换为**高亮呼吸态**。

- **重录**：再次点击中间录音键，会重置“用户音频”并重新开始录制。

---

## 异常处理与防错 \(Edge Cases\)

- **无效录音**：时长 $<1s$ 或全静音，IP 提示“没听清，再大声点”，不点亮“下一题”，右侧按钮不激活。

- **静默引导**：阶段二后 10s 无操作，IP 挥手并重播 VO。

- **点击防抖**：所有按钮设置 `300ms` 防抖，防止音频重叠播放。

# 题型四深度规约：互动涂色组件 \(Type\_InstructionColoring\)

## 业务目标与教研映射

- **考核能力**：考察 3\-7 岁幼儿在发音清晰、语速缓慢的语音提示辅助下，听力理解基础日常指令（颜色词 \+ 物品/动物词）并做出对应精细动作操作的能力（对标 K2\-K3 听后涂色水平） 。  

- **设计原则**：严格遵循“选色 $\rightarrow$ 填色”的两步法操作强校验；通过高保真“动态画笔”和即时“颜色渲染”，让幼儿对操作结果产生清晰的物理感知。

---

## 视觉布局规范 \(Visual Layout Specification\)

### 2\.1 顶部区域 \(Top Area\)

- **返回按钮**：固定在左上角。点击弹出退出确认弹窗（包含“继续寻宝”和“下次再来”选项），防止幼儿误操作退出。

### 2\.2 中间核心区 \(Main Stimulus Area\) —— 元素深度细化

根据横屏比例，中间区域固定划分为**左侧线稿区（70%宽度）与右侧固定调色盘区（30%宽度）**。所有展示元素完全由本地配置按 ID 动态吐出，图片比例严格执行 **4:3**：

```Plain Text
|  |                                       |  |       (棕色底板)        |  |

|  +---------------------------------------+  +-------------------------+  |

+--------------------------------------------------------------------------+
```

#### **左侧：黑白线稿区 \(Line Art Area\)**

- **素材规格**：标准的 SVG 矢量黑白线稿图。

- **包含元素**：线稿内包含多个独立且已切分 Path 的封闭区域，Path 的封闭区域分为不需要涂色Path 和需要涂色Path，不需要涂色Path填充指定的颜色，需要涂色Path，默认填充色为空（`fill: #ffffff`），带黑色勾边。根据题目ID判断，题目包括哪些元素，哪些是不需要涂色Path以及对应的颜色，哪些是需要涂色Path

- **“我的作品” \(User Masterpiece\)**：当幼儿成功执行填色后，对应的 SVG 封闭区域（Path）瞬间由黑白转为对应的彩色（如粉色花朵、黄色泰迪熊），这种实时的色彩渲染状态即为幼儿的动态作品呈现 。  

#### **右侧：固定调色盘区 \(Palette Area\)**

- **视觉装饰**：右侧整体承载于一块固定的**粉色底板**上 。  

- **包含元素及解释**：

    1. **原有音频（指令音）**：由调色盘左上角的绿色音量喇叭图标承载 。自动播放两遍核心涂色指令（如 Q4 播放 "Colour the flower pink\."，Q5 播放 "Colour the teddy bear yellow\."） 。  

    2. **画笔图标 \(Paintbrush\)**：位于喇叭图标右侧 。画笔的笔尖刷头颜色属于动态属性，其色值与当前用户选中的颜料完全同步（初始为无色） 。  

    3. **颜料盘按钮 \(Color Nodes\)**：纵向等距排列的圆形颜色饼，带轻微立体投影。根据题目ID确认配置的颜色 一般是3 种颜色：比如粉色（Pink）、黑色（Black）、黄色（Yellow） 。  

#### 题目举例：

- 题目要求：把花涂成粉色，算作答对

- 左侧：黑白线稿区 

    - 不需要涂色Path：  \[太阳\]  \[大树\]

- 右侧：固定调色盘区 

    - 喇叭：Colour the flower pink

    - 调色盘：粉色、黑色、黄色

- **右下角**：**右下角**：“下一题”导航按钮。基础包/增强包最后一题，则为“提交”按钮。

## 交互状态机 \(Interaction State Machine\)

### 阶段一：入场与锁定态 \(Entering \& Locked\)

- **动作**：组件加载，左下角 IP 播放画外音 \(VO\)（“轮到你创作啦！请你根据语音指令给图片涂色\.\.\.”） 。  

- **约束**：VO 播放期间，左侧线稿、右侧调色盘及右下角导航按钮全部处于锁定状态（`pointer-events: none`），拦截幼儿一切狂点操作。

### 阶段二：待机与提示态 \(Idle \& Prompting\)

- **动作**：画外音播放结束。

- **核心逻辑**：自动触发右侧喇叭的原有音频（指令音）播放，连续播放 **2 遍** 后停止 。播放中喇叭伴随动态波纹效果。  

### 阶段三：作答态 \(Coloring Action\)

- **动作**：幼儿自由进行“选色 $\rightarrow$ 填色”的交互循环。

- **微动效反馈**：

    1. **点击颜料盘**：被点击的圆形颜色饼产生 `scale(1.15)` 弹簧回弹动效，右侧画笔图标刷头平滑转变为对应颜色 。  

    2. **点击线稿区域**：被点击的封闭 Path 区域执行 `fill` 属性替换，颜色平滑渲染，并伴随清脆的“水滴落盘”音效。

- **重涂与改色**：幼儿可随时点击右侧新颜色，重复点击左侧已着色区域进行覆盖，系统以最终呈现的“我的作品”画面为准。

### 阶段四：完成与放行态 \(Completed\)

- **完成信号**：不当场校验对错。系统检测到幼儿只要在线稿区完成过**至少一次**有效的填色动作，右下角的全局“下一题”按钮立即由置灰切换为**高亮呼吸态**。

- **跳转**：点击“下一题”时，前端静态捕获当前所有 Path ID 的颜色数据，打包发送至后端，并执行过场跳转。

---

## 异常处理与防错 \(Edge Cases\)

- **未选色点击拦截**：若幼儿在阶段三尚未点击任何颜料盘（画笔为无色）时，直接狂点左侧线稿，线稿不产生颜色变化。此时右侧调色盘整体产生左右摇晃（Shake）的错误提示动效，IP 角色播放语音提示：“小画家，先选一个好看的颜料吧！”

- **静默超时引导**：若阶段二音频播放完毕后，幼儿停留在页面超过 10 秒无任何着色动作，IP 角色执行挥手动作，并重新单遍播放核心指令原有音频。

- **暴力快速连击防抖**：由于幼儿极易双手掌同时拍打屏幕，对线稿各 Path 区域的点击事件必须增加 `300ms` 的防抖限制，确保同一时间轴上只响应单路径的颜色变动，防止前端内存崩溃或 DOM 渲染异常。

- **阶段性结算**：完成 Q4\-Q5 全部涂色题后，系统自动触发童声欢呼音效“任务完成啦！”，屏幕飞出大拇指，并在中央弹出高保真的“小画家”奖章 。

# 第四章：全量页面级需求与流转规约

## 核心图层架构原则 \(Layering Principle\)

所有涉及答题和过场的页面，严格执行三层图层堆叠：

1. **底阶层 \(Layer 0 \- Background\)**：全屏渲染当前关卡专属的静态背景图或全景长图。

2. **中阶层 \(Layer 1 \- Stimulus \& Components\)**：核心交互区。题型组件（判断、选择、录音、涂色）采用透明背景，直接压在 Layer 0 之上。

3. **顶阶层 \(Layer 2 \- Global Navigation \& IP\)**：左上角返回键、右下角下一题键、左下角 IP 角色（外研小测，IP大小和下一题按钮相近）。

---

这是一个非常敏锐且关键的产品补漏！既然我们决定脱离原有的“小初版”系统，采用纯 H5 独立部署，那么**除了“手机号\+验证码\+三项协议”的基础登录弹窗外，我们还遗漏了一个极其核心的业务阻断点：【学段信息的采集】**。

在 V1\.0 的 PRD 规划中，明确要求需要收集幼儿的四个学段：小班（3\-4岁）、中班（4\-5岁）、大班（5\-6岁）、幼小衔接（6\-7岁）。由于自适应引擎在第一关抽题时高度依赖这个年龄段特征，因此这个采集动作必须无缝融入到首次登录的链路中。

## 一、登录页

### 独立 H5 登录全链路流转逻辑 \(Login Flow\)

为了保证“森林寻宝”的沉浸感，全链路不发生物理页面的跳转（不白屏），全部采用 Layer 2（顶阶层）悬浮弹窗实现拦截与放行。

4. 后端校验该手机号绑定的 UID：

5. 选择学段后 $\rightarrow$ 弹窗销毁，进入【地图页】。

### 后端防刷与安全规约 \(Security Specifications\)

由于 H5 暴露在公网，极易遭遇短信盗刷，必须向后端开发强调以下安全指标：

1. **短信防刷机制 \(SMS Rate Limiting\)**：

    - 同手机号：1 分钟内限发 1 次，自然日限发 10 次。

    - 同 IP 限制：同一 IP 每小时调用发送短信接口不得超过 30 次。

## 八、全局全自动流转控制流程 \(Global Process Flow\)

整个系统通过全局状态机严格控制页面的生命周期。为了保证幼儿在横屏高交互场景下的沉浸感，系统严格执行“底阶背景层（Layer 0）平滑切换/留存，上层交互组件（Layer 1/2）动态挂载/销毁”的丝滑流转机制。

## 【阶段一：初始入场动线】

1. **【首页】**（背景：森林寻宝全景） $\rightarrow$ 幼儿点击吸底居中按钮“开始森林寻宝”。

2. 按钮触发弹簧动效，页面全屏淡出（Slide\-left，耗时 `400ms`） $\rightarrow$ 进入 **【地图页】**。

3. **【地图页】**（背景：卷轴羊皮纸森林地图） $\rightarrow$ 幼儿点击高亮发光的“森林大门”节点 $\rightarrow$ 触发金色粒子涟漪，页面向右横向平移（耗时 `600ms`） $\rightarrow$ 进入 **【第一关导言页】**。

---

## 【阶段二：基础包循环答题动线】

1. **【第一关导言页】**（背景：森林之门特写）：

    - 静态展示关卡名与核心标语，播放“外研小测”任务引入说明 VO（停留 `2.5s`）。

    - **无缝切题**：导言文字淡出，【森林之门】背景图留存（Layer 0 不动），上层（Layer 1）瞬间挂载 **【判断题组件】** 顺次作答。

2. **【跨关卡转场流转】**：

    - 当第一关（Q1\-Q8）全部顺次作答完毕，最后一题点击“下一题” $\\rightarrow$ 触发 **【关卡过场组件】**（顶部进度条激活，“外研小测”在进度条上向右奔跑，耗时 `1.2s`）。

    - 奔跑期间，Layer 0 静态背景图平滑横向卷轴向左滚动，进入下一关【地图页】。

    - 【地图页】结束 $\\rightarrow$ 挂载 **【第二关导言页】**（停留 `2.5s`） $\\rightarrow$ 文字淡出，上层挂载 **【智能录音口语组件】**。

3. **【同理流转第三关】**：

    - 第二关全部完成 $\\rightarrow$ 过场组件奔跑转场 $\\rightarrow$ 背景平滑切换为【精灵谷】 $\\rightarrow$ 挂载 **【第三关导言页】** $\\rightarrow$ 上层挂载 **【标准化选择题组件】** 顺次作答。

---

## 【阶段三：中场自适应判定分流阀门】

当幼儿在基础包第三关最后一题（第 22 题）点击“下一题”时，前端挂起强同步Loading，通过 1\.2s 的转场动画掩盖后端算分延迟。根据后端返回的正确率，流转状态机产生**两个完全相反的绝对分流走向**：

### 分支 A：正确率 $\ge 80\%$ ── 触发【森林游乐场过渡动线】

1. **剧情揭晓**：分流确定，页面瞬间爆发出漫天彩带粒子烟花（Confetti Blast），BGM 切换为高亢的铜管乐嘉年华音乐，进入 **【森林游乐场过渡页】**。

2. **开箱飞票剧情定格**：页面背景切为明亮活泼的嘉年华清晨（Layer 0）。中央的【森林宝箱】轰然开启，喷洒出大片宝石特效，一张纯星星流光装饰的【金色嘉年华入场券】从宝箱内平滑飞出，悬浮于高光下。

3. **告知与解锁**：霓虹木板标语亮起：“开启森林游乐场！高级挑战解锁！”，左下角“外研小测”换上马戏团团长礼服吹喇叭。

4. **点击放行**：幼儿点击【出发！进入嘉年华】 $\rightarrow$ 触发“大幕拉开”的全屏转场动效（耗时 `600ms`） $\rightarrow$ 切换底阶背景为黑白线条的【魔法彩绘园】 $\rightarrow$ 进入 **【第四关导言页】**，开启增强包（四、五、六关）挑战。

### 分支 B：正确率 $< 80\%$ ── 触发【基础包熔断退出动线】

1. **提前收尾**：分流确定，页面不跳出，不给予任何负面压力，执行全屏平滑淡入动效 $\rightarrow$ 进入 **【基础包完成页】**。

2. **基础荣誉定格**：页面背景切换为温暖的午后森林空地（Layer 0）。中央悬浮一个已经打开、装满宝石的黄金大宝箱。下方木板标语显示：“森林寻宝大成功！获得荣誉勋章！”，“外研小测”在左下角抱着蓝色大水晶微笑挥手。

3. **看报告跳转**：此时幼儿完成任务，将设备交还给家长。家长点击吸底居中高亮按钮【查看我的定级报告】 $\rightarrow$ 页面全屏安全淡出 $\rightarrow$ 挂载 **【双能级测评报告页 \(K1\-K2 启航探索级\)】**，流程闭环。

---

## 【阶段四：增强包终极通关动线】

1. 顺次进入分支 A 的幼儿，将顺次挑战完成第四关（互动涂色）、第五关（情境问答）、第六关（终极绘本判定）。

2. 当增强包第六关最后一题点击“提交/下一题”瞬间 $\rightarrow$ 页面触发终极通关结算 $\rightarrow$ 进入 **【增强包完成页】**。

# 第五章、全局公共交互组件与通用弹窗规约 \(Component\_GlobalModals\)

所有通用弹窗作为 **Layer 2（顶阶层）** 悬浮于当前页面之上，背景统一叠加 `rgba(0, 0, 0, 0.6)` 的半透明暗色遮罩（Mask），拦截下层一切点击事件。

## 1\.1 【弹窗 A】退出确认弹窗 \(Exit\_Confirmation\_Modal\)

- **触发场景**：幼儿在任何页面（地图页、关卡页、答题中）点击左上角【返回按钮】时触发。

- **UI 布局参数**：

    - **底板**：居中方形圆角白底板（圆角 `40px`，尺寸 `50vw * 60vh`），边缘带森林藤蔓装饰。

    - **中间文案**：黑色粗体字：“宝贝，真的要离开吗？宝藏就在前面啦！”

    - **IP 动效**：IP 角色（外研小测）在弹窗内做出两眼泪汪汪的挽留动画。

- **双键控制逻辑**：

    - **左侧按钮**：【下次再来】。浅灰色矩形圆角按钮，点击后清除当前临时答题状态，调用微信原生 API 或路由直接关闭小程序/H5。

    - **右侧按钮**：【继续寻宝】。高亮橙色呼吸按钮，点击后弹窗执行 `Scale(1 -> 0)` 销毁，下层页面状态无缝恢复，音视频继续播放。

## 1\.2 【弹窗 B】麦克风权限引导弹窗 \(Mic\_Permission\_Modal\)

- **触发场景**：在题型三（口语录音题）中，系统调用设备麦克风 API 返回 `fail auth deny`（用户拒绝授权）时触发。

- **UI 布局参数**：

    - **底板**：标准白底板，无复杂装饰。

    - **核心文案**：**“哎呀，外研小测听不到你的声音啦！请爸爸妈妈帮忙打开麦克风权限哦\~”**

    - **操作指引图**：弹窗中央展示一张 1:1 的手机系统设置截图，用动态红色圆圈圈出“允许麦克风访问”的位置。

- **单键控制逻辑**：

    - **吸底按钮**：【去开启权限】。高亮绿色按钮，点击后直接唤起微信小程序的系统权限设置页（`wx.openSetting`），不亮起右下角“下一题”。

## 1\.3 【弹窗 C】网络断线/超时重试弹窗 \(Network\_Retry\_Modal\)

- **触发场景**：在关卡切换、中场算分或最终报告生成时，API 请求响应超时（\>5秒）或返回 5xx 错误时触发。

- **UI 布局参数**：

    - **文案**：**“信号被大树挡住啦！外研小测正在努力连接中\.\.\.”**

    - **动作按钮**：【重新寻找宝藏】。

- **异常防错机制**：点击按钮后，前端静默触发原接口的 `Axios Retry` 机制。若重试 3 次依然失败，弹窗切换为最终兜底态：“宝藏箱盖子卡住了，请检查网络后再试一次吧”。

# 第六章：前端静态资源预加载与性能规约 \(Performance\_Preload\_Specs\)

由于幼儿的专注力极易被页面加载时的“白屏”、“菊花图（Loading旋转）”和“音频延迟”打断，本系统必须执行**零延迟（Zero\-Lag）预加载策略**。

## 1\.1 资源分级预加载状态机

系统利用幼儿处于【关卡导言页（停留2\.5s）】和【作答中】的碎片时间，在后台静默拉取后续资源：

```Plain Text
[ 用户处于：首页 ] ──> 异步预加载 ──> 基础包地图页背景、IP开场VO
       │
[ 用户处于：地图页 ] ──> 异步预加载 ──> Stage 1 导言页背景、Q1-Q2的所有4:3图片
       │
[ 用户处于：Stage 1 答题中 ] ──> 每做完一题，静默预拉取下一题的 itemAudioUrl（题干音频）
       │
[ 用户处于：Stage 3 答题中 ] ──> 从第20题开始，提前前端仅预加载【基础包完成页】和 【森林游乐场过渡页】的 Layer 0 纯背景图。
```

## 1\.2 核心性能指标约束 \(KPI Standards\)

开发团队在进行组件代码审核时，必须严格达成以下技术指标：

- **音频播放延迟 \(Audio Latency\)**：点击左上角喇叭图标或触发自动播放时，音频文件的解码至播放响应时间必须 $\le$** 150ms**。必须使用 Web Audio API 或音频上下文池（Audio Context Pool）常驻内存，杜绝“点击喇叭后卡顿 1 秒才发音”的现象。

- **图片渲染防抖 \(Image Rendering\)**：所有 4:3 题干和选项图片必须在当前题渲染前完成本地缓存。严禁出现卡片边框渲染完毕、但中间图片从上至下一点点刷出的“残缺视觉”。

- **动态动画帧率 \(Animation FPS\)**：IP 角色的骨骼动画（Lottie / WebGL）及转场横向平移，在 iPhone 16 Pro 及同等性能设备上必须稳定在 **60fps**，不触发掉帧卡顿。

# 核心解耦策略：试卷组（Pack Group）的实现方案

### 结论：打包与随机抽组的逻辑，必须在【独立后端的业务层】实现。

原有的题库系统只需要保持纯净，继续扮演“提供单关试卷内容”的角色。我们在独立后端的数据库中，引入一张【试卷组关系映射表】。通过这张表，后端不仅能把 3 套试卷“拼”成一个大组，还能在用户开始测评时全自动执行随机抽取。

## 一、 产品定位与业务目标

- **产品定位**：专为《外研定级测·幼儿版》打造的轻量级、独立化内容管理与自适应策略配置中心。

- **业务目标**：

    1. **资产标准化**：为教研团队提供零代码门槛的录入界面，统一管理 4:3 高清素材、音频及标准答案。

    2. **策略解耦**：将题库与自适应抽组逻辑解绑，支持教研随时在后台灵活更换 A1/A2、B1/B2 组的试卷内容，前台自动生效，无需研发发版。

    3. **学情沉淀**：精准记录幼儿的全量作答足迹，支持 SVG 涂色作品的静态化复原，为 25 级能力体系提供底层测评数据支撑。

## 二、 核心功能模块规约

### 模块一：原子题库管理 \(Atomic Question Bank\)

**功能描述**：用于单道题目的增删改查。所有进入系统的题目必须在此打碎为“原子态”。

### 模块二：单关试卷装配 \(Stage Paper Assembly\)

**功能描述**：将录入好的原子题目，像拼积木一样拼装成一张对应“单一固定关卡”的试卷（对应数据库 `t_toddler_paper`）。

- **列表页**：展示已有试卷（试卷 ID、试卷名称、包含题量、创建时间）。

- **组卷交互页 \(核心\)**：

    - **基础信息配置**：输入试卷 ID（如 `PAPER_101`）与试卷名称。

    - **题目穿梭框 \(Transfer / Drag\-and\-Drop\)**：

        - 左侧列表显示【原子题库】中未选择的题目（支持按题型、标签搜索）。

        - 右侧列表显示【已加入本试卷】的题目。

    - **顺次排序功能**：教研在右侧列表中，可以通过**上下拖拽**的方式，直接决定这批题目在游戏关卡中的真实作答顺序（即写入底层 `sort_order` 字段）。

    - 试题需要分组，同组的试题可以配置一个画外音

### 模块三：自适应大包组装 \(Adaptive Pack Configuration\)

**功能描述**：管理基础包（必做）与增强包（拔高）的试卷挂载映射关系，控制分流引擎的弹药库。

- **基础包配置区 \(Base Pack\)**：

    - 提供 A1 组、A2 组的配置卡片。

    - 每组卡片内固定包含三个插槽：【第一关：森林大门】、【第二关：拼图营地】、【第三关：精灵谷】。

    - 操作：点击插槽，弹出试卷选择器，将模块二中拼好的试卷挂载进去。

- **增强包配置区 \(Advanced Pack\)**：

    - 提供 B1 组、B2 组的配置卡片。

    - 插槽为：【第四关：彩绘园】、【第五关：森林剧场】、【第六关：宝藏终点】。

- **业务逻辑闭环**：此处的配置直接写入 `t_toddler_pack_group`。一旦保存，前端发起测评时的 Random 抽题算法将立即按照此处的最新映射进行下发。
